1. 先阅读 `docs/Personal-Learning-Plan-12-Months.md`，按月主题与每周任务建立个人学习计划。
2. 再根据你当前水平选择 `courses/` 中匹配的课程，按“基础→专业→前沿”顺序逐步学习。
3. 每个月完成一个小项目或实验，把结果写进 GitHub repo、报告和演示材料。
4. 同时参考 `docs/Personal-Career-Learning-Path.md` 规划证书、岗位方向和职业能力。

### 推荐首选学习顺序 / Recommended starting order
- 先学 `MAE101`、`MAE102`、`MAE103`，补基础机械制图、力学与传动原理。
- 之后进入 `MAE104`、`MAE105`，重点掌握控制理论、PLC、机电一体化与电气自动化。
- 再过渡到 `MAE302`、`MAE303`，学习智能制造、数字化、数控工艺与高级加工。
- 最后做 `MAE401` 毕业设计，结合前述知识完成一个综合样机或智能制造项目。

1. Start with `docs/Personal-Learning-Plan-12-Months.md` to build your own month-by-month study plan.
2. Choose matching course outlines from `courses/`, and follow the progression: Foundation → Core → Advanced.
3. Finish a small project or lab each month, and document your work in a GitHub repo, report, and demo.

### Recommended starting order
- Learn `MAE101`, `MAE102`, `MAE103` first for mechanical drawing, mechanics, and transmission fundamentals.
- Move to `MAE104` and `MAE105` next to master control theory, PLC, mechatronics, and electrical automation.
- Then cover `MAE302` and `MAE303` for intelligent manufacturing, digitalization, CAM, and advanced processing.
- Finally, complete `MAE401` Capstone Project as an integrated demonstration of your skills.

## 个人学习最佳实践 / Personal study best practices

- 每周复盘一次，记录“本周学到什么”、“遇到什么问题”、“下周要解决什么”。
- 每月完成一个可演示的小项目，并把代码、报告、视频留在 GitHub 或笔记中。
- 优先把每个阶段的成果变成可展示的 demo，而不是只做笔记。
- 结合证书目标安排学习时间，例如先考低压电工，再考 PLC 认证。
- 通过经验积累，逐步从“单项技能”转向“系统集成与调试能力”。

- Review weekly: note what you learned, what issues you faced, and what you will solve next week.
- Deliver one demonstrable mini-project per month, and keep code, reports, and videos in GitHub or notes.
- Focus on building demos rather than only taking notes.
- Align study time with certification goals, for example low-voltage electrician first, then PLC certification.
- Progress from one technical skill at a time toward system integration and debugging capability.
